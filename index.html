TEALIUM QUALITY ASSURANCE WEBSITE

 
 

Publish Testing


Test the Publish engine using this test page.Publish generates number of utag files.The most important is the utag.js which is the core process.This is the important file for tagging to work.For each active tag, there will a utag.*.js which will be loaded. There are also other utag files such as utag.footer and utag.sub. All these files are accessible to this test page. Also check the tag loading adding with Load rules, mapping data sources and extensions




Webcompanion Testing


Use this test page for testing Web companion. Usually web companion doesn't load when the profile data is not up-to-date with the production data. Advantage of using this test page is, web companion will be loaded with any data on the profile. It is not necessary that it has to be exact match with production data. Load web companion on this page & add data to the test profile.




Privacy Manager Extension Testing


The Multi-Opt out widget displayed on the test page will be useful for privacy Manager testing. Here you can easily modify tealium opt-out settings.Add Privacy Manger extension on a test profile. Also setup the correct widget Id on the UI. Choose opt method as 'Opted out by default'& Visitor selection type as 'by Category' on the extensions settings. Modify and test the privacy options using this widget.




ServerToServer Testing


There are tags that fires on server side. Add serverToServer tags on a profile.Enable Server to Server option on the UI. Use this test page to make sure these tags are not loading/firing on the browser side. Also test the non ServerToServer tag loading on the client side. Rocket Fuel, Tealium Event stream, SiteCatalyst Mobile App are some of the ServerToServer tags.
